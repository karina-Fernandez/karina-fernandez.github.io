Temas Aprendidos

¿Cómo funciona la web?

La web funciona mediante un sistema cliente-servidor donde los navegadores (clientes) solicitan recursos a servidores web a través del protocolo HTTP/HTTPS. El proceso involucra la resolución DNS para convertir nombres de dominio en direcciones IP, permitiendo que los usuarios accedan a sitios web de manera intuitiva usando nombres fáciles de recordar en lugar de direcciones numéricas complejas.

Sistema DNS (Domain Name System)

El DNS es como la "guía telefónica" de internet, traduciendo nombres de dominio legibles (como google.com) en direcciones IP numéricas que las computadoras pueden entender. Este sistema jerárquico distribuido permite que millones de usuarios accedan a sitios web sin necesidad de memorizar direcciones IP complejas, facilitando la navegación y el acceso a recursos en línea.

Frontend - Desarrollo del Lado del Cliente

El frontend es la parte visible de una aplicación web con la que interactúan directamente los usuarios. Incluye tecnologías fundamentales como HTML (estructura), CSS (presentación) y JavaScript (interactividad), así como frameworks modernos como React, Vue.js, Angular. También abarca herramientas de desarrollo como bundlers (Webpack, Vite), preprocesadores CSS (Sass, Less) y librerías de componentes.

Backend - Desarrollo del Lado del Servidor

El backend es la parte del servidor que maneja la lógica de negocio, bases de datos, autenticación y APIs. Utiliza lenguajes como Node.js, Python, Java, PHP, C# y frameworks como Express.js, Django, Spring Boot, Laravel. Incluye sistemas de gestión de bases de datos (MySQL, PostgreSQL, MongoDB), servicios en la nube (AWS, Azure, Google Cloud) y herramientas de DevOps para despliegue y monitoreo.

Ejercicios de Laboratorio - Resultados

Ejercicio 01

Reconocer y utilizar Visual Studio Code

Objetivo: Familiarizarse con las áreas de la pantalla principal y aplicaciones de la barra de menús.

Exploré sistemáticamente las diferentes áreas de VSC: el explorador de archivos para navegación de proyectos, el editor principal con resaltado de sintaxis, el terminal integrado para comandos, y el panel de extensiones. Aprendí a personalizar el entorno de trabajo, configurar temas, y utilizar la barra de menús para acceder a funcionalidades avanzadas como depuración y control de versiones.

Ejercicio 02

Paleta de comandos y GitHub Codespaces

Objetivo: Activar la paleta de comandos y usar GitHub Codespaces.

Activé la paleta de comandos (Ctrl+Shift+P) y practiqué comandos esenciales como "Format Document", "Toggle Terminal", "Git: Commit". Experimenté con GitHub Codespaces, creando un entorno de desarrollo en la nube que permite programar desde cualquier dispositivo con navegador, entendiendo sus ventajas para colaboración remota y acceso universal al código.

Ejercicio 03

Terminal y comandos Git

Objetivo: Usar terminal integrada y comandos Git básicos.

Utilicé la terminal integrada para ejecutar comandos Git fundamentales: git init para inicializar repositorios, git add para preparar cambios, git commit para confirmar modificaciones, git checkout para cambiar ramas, y git push para subir cambios. Practiqué la creación de ramas con git branch y fusión con git merge.

Ejercicio 04

Emmet y fragmentos de código

Objetivo: Usar disparadores Emmet y crear fragmentos personalizados.

Aprendí a usar disparadores Emmet para generar código HTML/CSS rápidamente: div.container>ul>li*5 para estructuras complejas, ! para plantillas HTML5. Creé fragmentos de código personalizados para estructuras repetitivas como componentes React, configuraciones de CSS Grid, y plantillas de funciones, aumentando significativamente la velocidad de desarrollo.

Ejercicio 05

Keyboard shortcuts en VSC

Objetivo: Dominar atajos de teclado para aumentar productividad.

Memoricé y practiqué atajos esenciales: Ctrl+D para selección múltiple de palabras idénticas, Ctrl+Shift+L para seleccionar todas las ocurrencias, Alt+↑/↓ para mover líneas, Ctrl+Shift+K para eliminar líneas, Ctrl+/ para comentar código. Estos atajos mejoraron dramáticamente mi velocidad de codificación y eficiencia en la edición de código.
