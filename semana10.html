Desarrollo de Aplicaciones Web con JSP, Jakarta EE y Spri

El desarrollo de aplicaciones web modernas en Java se apoya en tecnologías como JSP, Jakarta EE y el framework Spring. Estas herramientas permiten construir sistemas robustos, seguros y escalables, utilizando arquitecturas como MVC y principios como la inversión de control (IoC) y la inyección de dependencias.

Teoría Resumida con Conceptos Clave

  • • Apache Tomcat: Servidor web que funciona como contenedor de servlets y páginas JSP.
  • • JSP (Java Server Pages): Tecnología que permite generar contenido dinámico del lado del servidor utilizando sintaxis Java incrustada en HTML.
  • • Objetos implícitos: Variables como request, response, session y application que facilitan el manejo de datos en JSP.

Arquitectura MVC

  • • Modelo-Vista-Controlador: Patrón que divide la aplicación en capas. La vista presenta los datos, el controlador los procesa, y el modelo se encarga de la lógica y los datos.

    • Maven y Gradle

  • • Maven: Gestor de proyectos que automatiza la compilación, pruebas y empaquetado. Usa el archivo pom.xml.
  • • Gradle: Alternativa moderna y flexible a Maven, usa scripts Groovy.

    • Jakarta EE

  • • Plataforma para desarrollar aplicaciones empresariales en Java. Incluye soporte para servlets, JSP, seguridad, persistencia, entre otros.

    • Spring Framework

  • • Framework para aplicaciones Java empresariales.
  • • IoC (Inversión de Control): El framework gestiona la creación y ciclo de vida de los objetos.
  • • DI (Inyección de Dependencias): Las dependencias se inyectan automáticamente en lugar de ser instanciadas manualmente.
  • • Spring Boot: Simplifica la configuración y despliegue de aplicaciones Spring.

    • Desarrollo de APIs REST con Spring Boot

  • • Se crea una API utilizando controllers, models, repositories y services.
  • • Se configura la conexión con MySQL.
  • • Se pueden realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) y otras como filtros, paginación y validaciones.
